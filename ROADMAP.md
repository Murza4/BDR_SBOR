# Roadmap

## Следующий production-этап

### Multi-user auth

- Добавить таблицы пользователей, ролей и активных сессий.
- Хранить password hash в БД через современный алгоритм, например Argon2id или bcrypt.
- Добавить роли минимум: администратор, оператор, только просмотр.
- Привязать import-сессии к пользователю.
- Ограничить просмотр и скачивание результатов владельцем сессии или администратором.

### Ротация и инвалидация сессий

- Хранить серверные session id в SQLite/PostgreSQL.
- Добавить logout с серверной инвалидацией.
- Добавить принудительную ротацию session id после входа.
- Добавить TTL и очистку старых auth-сессий.
- Добавить возможность сбросить все активные сессии пользователя.

### Централизованные фоновые задачи

- Вынести распознавание и сборку в отдельный worker-процесс.
- Использовать Redis/RQ, Celery или Dramatiq.
- Хранить статус задач в БД.
- Добавить retry policy, таймауты и ограничение параллельности.
- Добавить отдельный экран статуса задачи с прогрессом по файлам.

### Browser e2e в CI

- Расширить smoke e2e до полного сценария:
  загрузка тестовых Excel-файлов, ожидание распознавания, подтверждение маппинга, сборка и скачивание результата.
- Добавить проверку auth-flow: login, protected redirect, logout.
- Добавить проверку ошибок: неподдержанный формат, слишком большой файл, неверный CSRF.
- Сохранять screenshot/video artifacts при падении e2e.

### Production storage

- Перевести SQLite на PostgreSQL для сетевого режима.
- Настроить backup/restore БД.
- Настроить отдельное хранилище файлов, например S3-compatible storage с server-side encryption.
- Добавить явную политику retention per project/user.
