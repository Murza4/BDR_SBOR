# Мастер сборки БДР

Локальное web-приложение для загрузки до 7 Excel-файлов, анализа структуры workbook, подтверждения ролей, ручного маппинга и сборки итогового файла БДР.

## Что уже реализовано

- Загрузка от 1 до 7 файлов `.xlsx` или `.xlsm`.
- Создание сессии импорта и сохранение исходников в `data/uploads`.
- Анализ листов, размеров, плотности данных, заголовков, формул, таблиц и повторяющихся структур.
- Fingerprint workbook и поиск похожих файлов.
- Автоопределение роли файла: основной, маппинг, справочник, шаблон, прочее.
- Предпросмотр с ручной корректировкой ролей.
- Fuzzy-маппинг полей БДР по русским синонимам.
- Ручное подтверждение сопоставлений.
- Сборка итогового Excel-файла `itogovyj_bdr.xlsx`.
- Отчет с ошибками, предупреждениями, статистикой и журналом решений.
- Ускоренный первичный анализ: `read_only=True`, ограничение зоны чтения, кэш fingerprint и параллельная обработка файлов.
- Отчет производительности: время стадий, число прочитанных строк и ячеек, попадание или промах кэша и количество параллельных задач.
- Единый Excel-loader для чтения и создания workbook через `openpyxl`.
- Отдельный compare-слой для поиска похожих workbook, листов, заголовков и формул.
- Отдельный workflow-слой для wizard-сценария: загрузка → распознавание → предпросмотр → маппинг → сборка.
- Тесты для анализа, маппинга, сборки и валидации загрузки.
- Защитные ограничения на размер файла, распакованный объем Excel-архива, число листов и число строк при сборке.
- Нейтрализация значений, похожих на Excel-формулы, перед записью в итоговый файл.
- Авторизация администратора через подписанную HttpOnly-cookie.
- Фоновая очередь распознавания workbook после загрузки.
- Шифрование исходников, отчетов и итоговых файлов на диске.
- Автоочистка старых uploads, outputs, reports, cache и SQLite-сессий.

## Архитектура

Код разделен по слоям:

- `src/bdr_wizard/ui` — локальный web-мастер на FastAPI и Jinja, только пользовательский поток и шаблоны.
- `src/bdr_wizard/auth.py` — простая авторизация администратора без внешней БД.
- `src/bdr_wizard/ingestion` — прием и сохранение Excel-файлов.
- `src/bdr_wizard/security.py` — лимиты, проверки и нейтрализация опасных Excel-значений.
- `src/bdr_wizard/storage/encryption.py` — прозрачное Fernet-шифрование файлов на диске.
- `src/bdr_wizard/excel/loader.py` — единая точка открытия и создания workbook через `openpyxl`.
- `src/bdr_wizard/excel/analyzer.py` — structural screening, заголовки, формулы, fingerprint и автоопределение роли.
- `src/bdr_wizard/excel/compare.py` — сравнение workbook по метаданным, листам, заголовкам и формулам.
- `src/bdr_wizard/analyzer` — совместимый фасад для старых импортов.
- `src/bdr_wizard/mapper` — сопоставление колонок и сохранение правил.
- `src/bdr_wizard/builder` — сборка итогового файла БДР.
- `src/bdr_wizard/workflow/wizard.py` — orchestration wizard-сценария без UI-логики.
- `src/bdr_wizard/workflow/jobs.py` — фоновая очередь задач распознавания.
- `src/bdr_wizard/workflow/builder.py` — совместимый workflow-вход для сборки БДР.
- `src/bdr_wizard/reporting` — audit trail и отчет.
- `src/bdr_wizard/storage` — SQLite-хранилище сессий.

## Ускорение обработки

Первичный анализ workbook выполняется как structural screening:

- входные книги открываются только через `WorkbookLoader.open_for_analysis(...)`;
- внутри loader используется `openpyxl.load_workbook(..., read_only=True, keep_links=False)`;
- workbook закрывается после обработки;
- читается только ограниченная used range-зона: первые строки для заголовков и ограниченное число строк/колонок для структурной сигнатуры;
- скрытые и пустые листы отмечаются и не попадают в углубленную обработку;
- fingerprint workbook кэшируется в `data/cache/workbooks`;
- независимые файлы анализируются параллельно на уровне целых файлов;
- итоговый отчет содержит раздел `производительность`.

Углубленная проверка запускается только для данных, выбранных правилами маппинга. При сборке БДР исходные листы также читаются через `WorkbookLoader.open_for_build(...)` в read-only режиме, а собранный DataFrame проходит агрегированную Pandera-валидацию.

## Защита при обработке Excel

- Максимальный размер одного загружаемого файла: 50 МБ.
- Максимальный суммарный размер файлов в одной сессии: 200 МБ.
- Перед открытием workbook выполняется проверка ZIP-структуры `.xlsx` / `.xlsm`: число внутренних элементов, распакованный объем и коэффициент сжатия.
- Книги с чрезмерным числом листов отклоняются до углубленной обработки.
- При сборке БДР читается ограниченное число строк, чтобы один огромный лист не перегрузил приложение.
- Значения, начинающиеся с `=`, `+`, `-`, `@` или управляющих переводов строки, записываются в итоговый Excel как текст.
- Если в исходной формульной ячейке нет сохраненного Excel-значения, формула переносится в итоговый файл как безопасный текст и отражается в предупреждениях.
- Builder использует только правила сопоставления, которые пользователь подтвердил на экране маппинга.
- POST-формы защищены локальным CSRF-токеном.
- По умолчанию web-приложение принимает запросы только с локального компьютера. Для осознанного сетевого запуска нужно задать `BDR_ALLOW_REMOTE=1`.
- Исходные Excel-файлы, итоговый БДР и JSON-отчеты хранятся как зашифрованные `.enc` файлы.
- Ключ шифрования берется из `BDR_ENCRYPTION_KEY`; если переменная не задана, создается локальный ключ `data/bdr_encryption.key`.
- Старые артефакты очищаются при старте приложения. Срок хранения по умолчанию — 30 дней, настройка: `BDR_RETENTION_DAYS`.

## Авторизация

По умолчанию вход включен. Учетные данные задаются переменными окружения:

```bash
export BDR_ADMIN_USER=admin
export BDR_ADMIN_PASSWORD='сложный-пароль'
export BDR_AUTH_SECRET='длинный-секрет-для-cookie'
```

Для локальной разработки авторизацию можно отключить:

```bash
export BDR_AUTH_DISABLED=1
```

Если переменные не заданы, используется локальная пара `admin` / `admin`; для production это нужно заменить перед запуском.

## Фоновые задачи

После загрузки файлы сохраняются, сессия ставится в очередь, а пользователь попадает на экран обработки. Количество worker-потоков задается переменной:

```bash
export BDR_JOB_WORKERS=2
```

## Границы ответственности

- UI не открывает Excel-файлы и не сравнивает workbook напрямую.
- Wizard-слой управляет этапами и вызывает backend-сервисы.
- Analyzer не сравнивает workbook между собой.
- CompareEngine работает только на уже извлеченных профилях workbook и не открывает Excel-файлы.
- Builder читает исходные листы только через `WorkbookLoader`.

## Референсы

При проектировании использованы идеи из:

- Django Data Wizard: пошаговый импорт, выбор источника, предпросмотр и подтверждение перед импортом.
- openpyxl: чтение и запись workbook, листов, ячеек и формул.
- ExcelCompare: сравнение workbook и отчетность по похожим структурам.

## Запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
uvicorn main:app --reload
```

После запуска откройте:

```text
http://127.0.0.1:8000
```

## Тесты

```bash
pytest
```

Для e2e-тестов через браузер:

```bash
pip install -e ".[dev,e2e]"
playwright install chromium
pytest tests/e2e
```

Для воспроизводимой установки текущие версии зафиксированы в `requirements.lock`.

## Важное правило локализации

Весь пользовательский слой должен быть на русском языке: экраны, кнопки, подсказки, предупреждения, ошибки, отчет и документация. Если в интерфейсе появляется английский текст, это считается ошибкой.
