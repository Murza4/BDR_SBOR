# Release notes

## 0.1.0-production-lite

Текущая версия фиксирует production-lite состояние локального мастера БДР.

### Реализовано

- Загрузка от 1 до 7 Excel-файлов `.xlsx` / `.xlsm`.
- Wizard-поток: загрузка, фоновое распознавание, предпросмотр, маппинг, сборка результата.
- Анализ workbook: листы, размеры, плотность данных, заголовки, формулы, fingerprint.
- Автоопределение ролей файлов и ручная корректировка.
- Fuzzy-маппинг по русским синонимам и ручное подтверждение правил.
- Сборка итогового Excel-файла БДР и JSON audit-отчета.
- Защитные лимиты: размер файла, суммарный размер сессии, ZIP-preflight, число листов, число строк сборки.
- Защита от Excel formula injection.
- Обработка формул без сохраненных cached values с явным предупреждением.
- Авторизация администратора через подписанную HttpOnly-cookie.
- CSRF-защита POST-форм.
- Local-only режим по умолчанию.
- Фоновая очередь распознавания после загрузки.
- Шифрование исходников, отчетов и итоговых файлов на диске.
- Автоочистка старых uploads, outputs, reports, cache и SQLite-сессий.
- Lock-файл зависимостей `requirements.lock`.
- Unit/integration тесты и browser smoke e2e-скелет.
- GitHub Actions workflow для тестов, dependency audit и browser smoke.

### Проверки

- `python -m compileall -q src`
- `pytest`
- `python -m pip check`
- `python -m pip_audit`

### Важные настройки production-lite

- `BDR_ADMIN_USER`
- `BDR_ADMIN_PASSWORD`
- `BDR_AUTH_SECRET`
- `BDR_ENCRYPTION_KEY`
- `BDR_RETENTION_DAYS`
- `BDR_JOB_WORKERS`
- `BDR_ALLOW_REMOTE=1` для осознанного сетевого запуска.

### Ограничения

- Модель пользователей пока single-admin, без ролей.
- Пароль администратора задается через переменную окружения, а не через БД.
- Сессии авторизации не хранятся централизованно и инвалидируются по TTL.
- Фоновая очередь локальная, внутри процесса приложения.
- E2E в CI покрывает smoke-сценарий, а не полный пользовательский путь с реальными Excel-файлами.
